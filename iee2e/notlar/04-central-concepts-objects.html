      📌 SAP'deki 4 Temel Nesne

      SAP sisteminde tüm yapı objects (nesneler) üzerinden kurulur:

      • Enterprise Structure → organizasyon yapısı (Client, Company Code, Plant)
      • Master Data → değişmeyen temel veriler
      • Transactional Data → günlük işlemler
      • Documents → bu işlemlerin kayıtları

      👉Ana ilişki:
      Enterprise Structure + Master Data + Transactional Data + Documents = SAP sisteminin tamamı
      🧠 2. TEMEL KAVRAMLAR (ÇOK ÖNEMLİ)
      🏢 Enterprise Structure

      Şirketin organizasyon yapısı

      • Client
      • Company Code
      • Plant
      👉Tüm veriler buna bağlıdır
      📚 Master Data (EN KRİTİK KONU)

      Tanım: Uzun süre değişmeyen veri

      İşin "nouns (isimleri)"

      • Customer
      • Product (Material Master)
      • Employee
      • Supplier
      👉Her işlemde tekrar veri girmemek
      👉Sistemin otomatik doldurması
      🔄 Transactional Data

      Tanım: Günlük işlemler

      İşin "verbs (aksiyonları)"

      • Sales order
      • Invoice
      • Payment
      • Shipment
      👉Sürekli oluşur ve değişir
      📄 Documents

      Tanım: İşlemlerin sistemde kaydı

      • Sales order document
      • Invoice document
      • Purchase order document
      ⚠️👉 Çok önemli: Her transaction → bir document oluşturur
      🔗 3. ARALARINDAKİ İLİŞKİ (EN KRİTİK KISIM)
      👉SAP mantığı şu:
      Master Data hazırdır → Kullanıcı işlem yapar (Transactional Data) → Sistem bunu Document olarak kaydeder → Tüm bunlar Enterprise Structure altında olur
      🎯 Örnek (EZBERLİK)

      Bir müşteri sipariş verir:

      Enterprise StructureBu işlem bir Company Code & Plant'e bağlıdır → Enterprise Structure

      processing time → plant'e özel olabilir

      📌 ⚡ 5. MASTER DATA NEDEN ÖNEMLİ?
      • İşlem hızını artırır
      • Hataları azaltır
      • Otomatik veri doldurma sağlar
      👉Eksik master data = yavaş sistem + hata
      📌 SAP Sistemleri Arası Entegrasyon

      SAP sadece tek sistem değildir — farklı çözümler birlikte çalışır:

      • SAP S/4HANA (core ERP)
      • SAP SuccessFactors (HR)
      • SAP Ariba (Procurement)

      👉Bu sistemlerin hepsi aynı master data'yı kullanır

      Bu yüzden:

      •Veriler sistemler arasında senkronize (synchronized) olmalı
      •Ve tek doğru kaynak (single source of truth) olmalı

      Master Data Synchronization — MDI (Master Data Integration) ve MDG (Master Data Governance)

      🧠 2. TEMEL KAVRAMLAR
      🔗 Master Data Synchronization

      Farklı SAP sistemleri arasında veri uyumu sağlama
      Aynı customer / product / employee her yerde aynı olmalı

      🔄 MDI (Master Data Integration)

      Tanım: Farklı sistemlerden gelen veriyi birleştirir

      Amaç: Tek ve güvenilir veri görünümü (single trusted view)

      🛡️ MDG (Master Data Governance)

      Tanım: Master data'nın doğruluğunu · kalitesini · güvenliğini kontrol eder (güvenliğini kontrol eder)

      Aynı zamanda:
      • veri kuralları
      • onay süreçleri (approval)
      • compliance yönetimi

      🔗 3. NASIL ÇALIŞIR? (ÇOK ÖNEMLİ)

      Örnek:

      •Employee data → SAP SuccessFactors
      •Supplier data → SAP Ariba
      •Financial data → SAP S/4HANA

      👉Bu veriler:
      •MDI ile birleşir
      •MDG ile kontrol edilir
      📌 ⚡ 4. NEDEN ÖNEMLİ?
      ❌ Olmadığında

      Eğer olmazsa:

      • Farklı sistemlerde farklı veri olur
      • Hatalar çıkar
      • Süreçler bozulur
      ✅ Olduğunda

      Eğer varsa:

      • Veri tutarlı (consistent) olur
      • Süreçler düzgün çalışır
      • Karar verme iyileşir
      🧠 Tek Cümlelik Ezber
      👉MDI = birleştir  |  MDG = yönet  |  Amaç: single source of truth
